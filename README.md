## 版本说明

版本规则：

RobotCtrl_vNUM1.NUM2.NUM3.NUM4_alpha（commit_id）

    1. 功能1添加NUM2++
    2. 功能2添加NUM2++
    1. BUG1修复NUM3++
    2. BUG2修复NUM3++

- NUM1代表大幅度框架升级
- NUM2代表新功能添加
- NUM3代表BUG修复个数
- NUM4代表发布月日
- commit_id代表发布时commit的id号

**注意首先填写功能添加列表和BUG修复列表，最后相应更新版本号。**
**NUM4为发布日期，不要存在太多的版本号，只有在发布，或者有必要更新版本号时才产生新的版本号**
**新产生的apk版本要备份到ftp中**
